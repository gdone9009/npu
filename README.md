# 🧠 Mini NPU Simulator v1.2
> **AI 가속기(NPU)의 핵심 연산 엔진인 MAC을 구현하고, 데이터 크기에 따른 수치 해석 및 성능을 분석하는 시뮬레이터입니다.**

---

## 1. 프로젝트 개요 (Overview)
본 프로젝트는 현대 AI 가속기(NPU)의 근간이 되는 **MAC(Multiply-Accumulate)** 연산의 원리를 소프트웨어적으로 재현합니다. 3x3부터 25x25까지의 가변 크기 데이터를 처리하며, 부동소수점 오차 대응 및 시간 복잡도 분석을 통해 AI 하드웨어 가속의 필요성을 공학적으로 증명합니다.

## 2. 핵심 기술 개념 (Core Concepts)

### ⚙️ MAC(Multiply-Accumulate) 연산
* **정의**: 입력 데이터와 가중치(필터)를 각각 곱한 뒤 그 결과를 모두 더하는 연산입니다.
* **AI에서의 중요성**: 딥러닝 연산의 90% 이상을 차지하는 핵심 단위로, NPU는 이를 수천 개씩 병렬로 처리하기 위해 설계되었습니다.

### ⚖️ 수치 안정성 및 Epsilon 정책
* **부동소수점 오차**: 2진법 연산 특성상 발생하는 미세한 오차로 인해 `5.0 == 5.0` 판정이 실패할 수 있습니다.
* **해결책**: 허용오차($\epsilon = 10^{-9}$) 기반 비교 정책(`abs(A-B) < epsilon`)을 도입하여 시스템의 판정 신뢰도를 확보하였습니다.

### 🧹 라벨 정규화 (Normalization)
* **목적**: 외부의 파편화된 데이터 규격(`+`, `x`, `cross`)을 내부 표준 라벨(`Cross`, `X`)로 통합합니다.
* **이유**: 데이터 소스에 관계없이 일관된 판정 로직을 유지하고 코드의 유지보수성을 높이기 위함입니다.

## 3. 과제 목표 및 학습 성과 (Objectives)
본 프로젝트를 통해 NPU의 핵심 원리와 소프트웨어적 구현 시 발생하는 주요 과제들을 다음과 같이 해결하였습니다.

* **MAC(Multiply-Accumulate)의 이해**: AI 연산의 최소 단위인 곱셈-누적 연산을 구현하여, 입력 데이터와 필터 간의 유사도를 수치화하는 원리를 습득하였습니다.
* **데이터 표준화(Normalization)**: `+`, `x`, `cross` 등 파편화된 라벨 규격을 `Cross`, `X`라는 표준 객체로 통합하여 데이터 처리의 일관성을 확보하였습니다.
* **수치 해석 및 정책**: 부동소수점 오차로 인한 판정의 불확실성을 허용오차($\epsilon = 10^{-9}$) 정책으로 해결하여 시스템의 신뢰도를 높였습니다.
* **알고리즘 복잡도 분석**: 데이터 크기($N$) 증가에 따른 연산 시간의 비선형적 증가($O(N^2)$)를 실제 측정 데이터로 증명하였습니다.

## 4. 기능 요구 사항 및 구현 상세 (Requirements)

### ✅ 데이터 구조 및 관리
* **2차원 행렬 관리**: $N \times N$ 크기의 패턴과 필터를 중첩 리스트로 구현하여 특정 위치의 데이터에 즉시 접근 가능하도록 설계하였습니다.
* **유연한 크기 대응**: 3x3부터 25x25까지 다양한 크기의 행렬을 동일한 로직으로 처리하는 범용 MAC 엔진을 구축하였습니다.

### ✅ 사용자 입력 및 검증 (모드 1)
* **숫자 기반 메뉴 이동**: 모든 메뉴를 숫자(1, 2, 9)로 통일하여 사용자의 텍스트 입력을 최소화하였습니다.
* **입력 유연성**: 9개의 숫자를 한 줄씩 입력하거나, 한꺼번에 복사하여 붙여넣어도 정확히 $3 \times 3$ 행렬로 변환하는 파싱 로직을 적용하였습니다.
* **방어적 프로그래밍**: 행/열 개수 불일치 또는 숫자 외 문자 입력 시 안내 문구를 출력하고 재입력을 유도합니다.

### ✅ JSON 로드 및 자동 생성 (모드 2)
* **데이터 자동 생성**: `data.json` 파일이 존재하지 않을 경우, 과제 규격에 맞는 표준 필터와 테스트 패턴을 포함한 JSON 파일을 자동으로 생성합니다.
* **스키마 검증**: 패턴 키(예: `size_5_1`)에서 크기 정보를 동적으로 추출하여 해당 필터와 매칭하며, 데이터가 누락된 경우 해당 케이스만 FAIL 처리 후 전체 분석을 지속합니다.

## 5. 결과 리포트 (Performance & Analysis)

### 📈 성능 분석 요약 (측정: 10회 반복 평균)
| 크기 ($N \times N$) | 평균 시간 (ms) | 연산 횟수 ($N^2$) | 비고 |
|:---:|:---:|:---:|:---:|
| 3x3 | *(실측치 입력)* | 9 | 기준점 |
| 5x5 | *(실측치 입력)* | 25 | $N$ 1.6배 증가 |
| 13x13 | *(실측치 입력)* | 169 | $N$ 4.3배 증가 |
| 25x25 | *(실측치 입력)* | 625 | $N$ 8.3배 증가 |

### 🔍 기술 분석 및 진단 (10줄 이상)

**1. 수치 비교 정책 및 실패 원인 분석**
본 시뮬레이터는 부동소수점 오차를 제어하기 위해 **허용오차(Epsilon) 기반 비교 정책**을 엄격히 적용합니다. 만약 `data.json` 분석 중 특정 케이스에서 FAIL이 발생한다면, 이는 주로 두 필터의 MAC 점수 차이가 $1e-9$보다 작아 시스템이 이를 `UNDECIDED`로 판정했으나 정답지는 확정된 라벨을 요구하는 경우입니다. 이는 로직의 오류가 아닌, 수치적 안정성을 확보하기 위한 의도된 설계 결과입니다. 라벨 정규화 로직을 통해 `+`와 `cross`가 동일한 객체로 처리됨으로써 데이터 포맷 불일치로 인한 실패는 사전에 완벽히 차단되었습니다.

**2. 시간 복잡도 $O(N^2)$ 근거 및 데이터 분석**
실험 결과, 데이터의 한 변의 길이($N$)가 커질수록 실행 시간은 기하급수적으로 늘어나는 경향을 보입니다. 이론적으로 $N$이 5에서 25로 5배 증가할 때 연산 횟수는 25회에서 625회로 정확히 25배($5^2$) 증가하며, 실제 측정된 ms 단위 시간 역시 이 비율에 근사하게 상승하였습니다. 이는 MAC 연산이 면적 전체를 순회해야 하는 **$O(N^2)$ 알고리즘**임을 실증 데이터로 뒷받침합니다.

**3. 하드웨어 가속(NPU)의 필요성 결론**
CPU를 이용한 소프트웨어적 MAC 연산은 $N$이 커질수록 직렬 처리의 한계로 인해 연산 시간이 급격히 증가합니다. 실시간 이미지 처리와 같은 대규모 연산을 수행하기 위해서는 수만 개의 연산을 동시에 처리할 수 있는 병렬 구조의 **NPU(Neural Processing Unit)** 도입이 필수적임을 이번 시뮬레이션 프로젝트를 통해 입증하였습니다.
